# Support

## How to Get Help

If you run into a problem while working through this course, here are the best ways to get support.

### Before Asking for Help

Check the following first:

1. Re-read the module section where you got stuck — the answer is often in the text.
2. Make sure your GitHub CLI is up to date: `gh --version`
3. Make sure you are authenticated: `gh auth status`
4. Check that your GitHub Copilot subscription is active at [github.com/settings/copilot](https://github.com/settings/copilot)

### Getting Help in This Repository

- **Questions about the course content** — Open a [Discussion](../../discussions) with a clear description of what you tried and what happened.
- **Errors or mistakes in the content** — Open an [Issue](../../issues/new) and label it `content-error`. Include the module name and a description of the issue.
- **Submission-related questions** — Comment on your own submission issue.

### What to Include When Asking for Help

When opening a discussion or issue, please include:

- Which module and section you are working on
- The exact command you ran
- The output or error message you received
- Your operating system and GitHub CLI version (`gh --version`)

The more context you provide, the faster someone can help you.
